三、Linux使用Docker安装Redis

docker pull redis

docker run -d --name redis --restart always -p 6379:6379 -v /usr/local/redis/config:/etc/redis -v /usr/local/redis/data:/data redis redis-server /etc/redis/redis.conf --requirepass "123456" --appendonly yes

查看容器状态

docker ps -a

发现没有启动成功，查看日志

docker logs eac7d27a5b15

原因是没有 redis.conf 文件，需要准备 redis.conf 文件放在指定路径下。从github下载文件到本地并上传

scp /Users/apple/Documents/redis.conf root@101.43.204.168:/usr/local/redis/config

如果提示没权限，可以运行下面命令再上传

ssh-keygen -R 101.43.204.168

重启发现仍然不能正常启动，查看日志

原因是配置文件版本不一致，根据提示，去github找到 6.2.6 版本的配置文件https://github.com/redis/redis/releases/tag/ 下载，将redis.conf 上传，重启

成功！

# 通过 Docker 命令进入 Redis 容器内部
docker exec -it redis /bin/bash

# 进入 Redis 控制台
redis-cli
# 通过密码进入Redis控制台
redis-cli -h 127.0.0.1 -p 6379 -a 123456

# 添加一个变量为 key 为 name , value 为 bella 的内容
> set name bella
# 查看 key 为 name 的 value 值
> get name

# 退出容器
exit
